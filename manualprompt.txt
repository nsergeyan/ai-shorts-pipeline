═══════════════════════════════════
RANDOM CATEGORY SELECTION
═══════════════════════════════════
Only choose topics that can be clearly shown through a short visual clip with obvious action or movement, and can be easily turned into three strong YouTube search queries.

Before choosing a topic, pick a random number from one to four. State the number first.

one → hidden lore mechanic
two → unexpected character detail
three → strange world rule
four → rare production or animation fact

Follow the selected category. Focus on ONE micro-detail, not a broad concept.

GOOD example: what the inside of a Titan's mouth smells like to its host
BAD example: how Titan transformation works

═══════════════════════════════════
WHAT MAKES A FACT GOOD (READ FIRST)
═══════════════════════════════════
The target is NOT "most obscure." The target is "makes a viewer say wait, WHAT?"

Obscurity is allowed and welcome — but ONLY when it raises the reaction. A rare-but-boring fact is worse than a mid-rarity fascinating one.

The ideal fact pattern (the "soap pattern"):
A small, concrete, almost ordinary question (a texture, smell, sound, habit, weird detail) applied to a famous or central thing — where the real, canon answer is funnier or stranger than expected.

Reference example (from another series, for calibration only):
"Sukuna's severed finger feels and smells like soap." → famous object, hidden physical detail, canon, zero lore needed, instant 'wait WHAT?'. THIS is the level and flavor to aim for.

═══════════════════════════════════
TOPIC DISCOVERY PROTOCOL (MANDATORY)
═══════════════════════════════════
Generate FIVE candidate ideas in the chosen category.
Each idea must name a SPECIFIC source anchor at this stage (chapter, episode, fanbook page, or interview) — vague ideas are discarded immediately.

For each candidate assign ONE score only:

WAIT-WHAT SCORE (one–five):
one → "ok, neat"
two → "mildly interesting"
three → "huh, I didn't know that"
four → "oh that's actually wild"
five → "wait, WHAT? — I want to tell someone right now"

A fact must score FOUR or higher to be used.

KILL TEST — automatically reject a candidate if it is:
- well-known to most fans (no surprise)
- obscure but DRY (stat numbers, IQ figures, voice-actor credits, staff names, release dates, episode-count records) — rare yet boring
- requires a lore lecture before the surprise lands
- philosophical interpretation or character summary
- more than forty percent similar to anything in the USED VIDEOS LIST

BANNED OVERSATURATED PATTERNS (the iceberg clichés — never pick these):
character IQ/stat numbers, voice-actor trivia, "the author originally planned…", hidden background text/languages, episode-count or runtime records, sleep/eating habits stated as a character's whole personality.

Pick the highest-scoring surviving candidate.
If no candidate scores four or higher, generate five NEW ideas.

═══════════════════════════════════
DUPLICATE DETECTION RULE
═══════════════════════════════════
Discard a topic if it matches any USED VIDEOS LIST entry on ANY of these:
- same character + same object / body part / habit
- same mechanic explained with different words
- same event reworded
- same "what does X feel/smell/taste/sound like" framing about the same subject

Example:
Existing: why gojo eats sweets
Do NOT generate: why gojo loves desserts

TIER DEFINITIONS
Tier one — CONFIRMED: episode number / manga chapter / fanbook page / creator interview

Tier three — THEORY: no strong source. Script MUST say "fans believe" or "one theory suggests."

RULE: Because this video must not exaggerate, prefer Tier one. Only use Tier three if the WAIT-WHAT score is five AND the script clearly flags it as a theory.

ELEVENLABS V3 AUDIO TAGS RULE:
Insert MINIMUM four [bracketed] performance tags inside the script to guide the AI voice.

Tag families you can draw from (invent your own variants freely):
- Emotional tone: [excited], [nervous], [frustrated], [calm], [sorrowful]
- Reactions: [sigh], [gulps], [whispers]
- Volume & energy: [whispering], [shouting], [quietly], [loudly]
- Pacing: [pauses], [stammers], [rushed], [hesitates]
- Tone cues: [cheerfully], [flatly], [deadpan], [playfully], [dramatic]
Tags can be layered: "[nervously] I... I'm not sure this works. [gulps] But let's try."

PUNCTUATION FOR DELIVERY:
Comma → short pause. Period → full stop. Question mark → questioning tone. Exclamation → emphasis. Ellipsis (...) → suspense pause.

NUMBER RULE: Never use digits. Write numbers as words (one, not 1).

WORD COUNT RULE:
Spoken text must be ninety to one hundred words. Do NOT count [bracketed] tags.
After writing, number each spoken word to confirm the count is ninety to one hundred, then output the clean version. If it fails, rewrite.

SCRIPT QUALITY CHECK (confirm all before output):
- ninety to one hundred words (tags excluded)
- contains one surprising fact (Wait-What four or higher)
- contains one light humorous moment
- ends on a curiosity hook
- includes four or more ElevenLabs tags
If any fail → rewrite.

VISUAL CLIP STRATEGY (TIKTOK)
═══════════════════════════════════
SCENE ACCURACY RULE: If the script references a specific moment, the visuals must come from that exact scene. Do not guess seasons or episodes — describe the scene itself.

YOUTUBE QUERY FORMAT (three queries):
Structure: [main subject] + Attack on Titan + [clear action/event]
Avoid: talking, explaining, discussion, vague actions.

SCENE QUERY VISUAL SEARCH PROTOCOL:
Gemini searches by literal physical description, not lore. Write the scene_query as pure visuals:
- Describe what the screen looks like: hair colors, distinct clothing, physical objects.
- Describe kinetic action: dashing, eyes widening, glowing aura, debris falling, steam rising.
- No lore jargon. Describe effects visually (e.g. "white steam pouring off a giant body", not "Titan regeneration").
- The query must match the most surprising visual in the script.
