STEP ZERO — HYPE CHECK (DO THIS FIRST, BEFORE ANYTHING ELSE)
═══════════════════════════════════
Before touching the random wheel, find out what is CURRENTLY hot in the [SERIES] fandom right now.

Run at least THREE web searches to detect current hype, such as:
- [SERIES] news [current month + year]
- [SERIES] trending / reddit / twitter this week

You are looking for a recent, real event that fans are reacting to RIGHT NOW: a finale, a new episode, a trailer, a movie reveal, a season announcement, an anniversary, a viral scene, an interview, etc.

DECISION:
- IF a genuinely current/hot topic exists (something released or trending recently and fans are actively talking about it):
  → The video MUST be anchored to that hot topic.
  → BUT you still must find ONE specific "wait-WHAT" micro-detail INSIDE that topic — not just "the finale aired" or "season two was announced." The hype gives you the SUBJECT; the rules below still force a surprising micro-fact.
  → Skip the random wheel. Go straight to TOPIC DISCOVERY using the hot topic as your anchor.

- IF nothing is genuinely trending (no real recent event, just old news):
  → Say "No current hype found — using random category."
  → Proceed to RANDOM CATEGORY SELECTION below.

CURRENCY RULE: Whatever topic you land on, confirm it has not been retconned, clarified, or expanded by any material released after the source you cite. Prefer the most recent canonical information.

WHAT MAKES A FACT GOOD (READ FIRST)
═══════════════════════════════════
The target is NOT "most obscure." The target is "makes a viewer say wait, WHAT?"

Obscurity is allowed and welcome — but ONLY when it raises the reaction. A rare-but-boring fact is worse than a mid-rarity fascinating one.

The ideal fact pattern (the "soap pattern"):
A small, concrete, almost ordinary question (a texture, smell, sound, habit, weird detail) applied to a famous or central thing — where the real, canon answer is funnier or stranger than expected.

Reference example (from the series Jujutsu Kaisen, for calibration only — do NOT reuse unless [SERIES] is that show):
"Sukuna's severed finger feels and smells like soap." → famous object, hidden physical detail, canon, zero lore needed, instant 'wait WHAT?'. THIS is the level and flavor to aim for, applied to [SERIES].

NOTE: Even when anchored to a hot/hype topic, the fact must still follow the soap pattern — a concrete micro-detail, not a plot summary of the new episode.

TOPIC DISCOVERY PROTOCOL (MANDATORY)
═══════════════════════════════════
Generate FIVE candidate ideas.
- If Step Zero found hype: all five candidates must connect to that hot topic.
- If using the fallback wheel: all five candidates must fit the chosen category.

Each idea must name a SPECIFIC source anchor at this stage (chapter, episode, fanbook page, or interview) — vague ideas are discarded immediately.

For each candidate assign ONE score only:

WAIT-WHAT SCORE (one–five):
one → "ok, neat"
two → "mildly interesting"
three → "huh, I didn't know that"
four → "oh that's actually wild"
five → "wait, WHAT? — I want to tell someone right now"

A fact must score FOUR or higher to be used.

KILL TEST — automatically reject a candidate if it is:
- well-known to most fans (no surprise)
- obscure but DRY (stat numbers, IQ figures, voice-actor credits, staff names, release dates, episode-count records) — rare yet boring
- requires a lore lecture before the surprise lands
- philosophical interpretation or character summary
- a character trait or personality quirk reworded as a surprising fact (e.g. "this character is actually really competitive" — this is characterization, not a fact)
- any fact whose source you could not open and read this session (memory-only citations are BANNED)
- more than forty percent similar to anything in the USED VIDEOS LIST

BANNED OVERSATURATED PATTERNS (the iceberg clichés — never pick these):
character IQ/stat numbers, voice-actor trivia, "the author originally planned…", hidden background text/languages, episode-count or runtime records, sleep/eating habits stated as a character's whole personality.

Pick the highest-scoring surviving candidate.
If no candidate scores four or higher, generate five NEW ideas.

VERIFICATION PROTOCOL (WEB SEARCH MANDATORY — DO BEFORE WRITING)
═══════════════════════════════════
You may NOT rely on memory for any factual claim. Memory is treated as Tier three (unverified) until confirmed by a live search this session.

For the single chosen candidate, you MUST:

STEP 1 — SEARCH: Run at least three web searches to verify the fact. Searches must target different source types:
  • the official wiki / fandom page for the specific character or object in [SERIES]
  • the original source (manga chapter or comic discussion, official artbook/fanbook scans/translations, or a creator interview)
  • a cross-check search phrased to DISPROVE the fact (e.g. "is [claim] actually canon" / "[claim] misconception")

STEP 2 — READ: Open and read the actual pages. Do not cite a result you only saw as a search snippet. If a wiki claim is used, follow it to ITS cited source and confirm that citation exists.

STEP 3 — DATE CHECK: Note the publish/update date of each source. If newer material could have changed or contradicted the fact, search specifically for that. Prefer the most recent canonical information.

STEP 4 — RECONCILE: If sources conflict, the fact is NOT Tier one. Either downgrade to a flagged theory or discard.

HARD RULE: If you cannot open and confirm a real source this session, you may NOT label the fact Tier one or Tier two. State that verification failed and pick another candidate.

TIER DEFINITIONS
Tier one — CONFIRMED: episode number / chapter / fanbook page / creator interview, confirmed by an opened source this session
Tier two — WIKI SOURCED: wiki article PLUS its own citation, both opened and confirmed this session
Tier three — THEORY: no strong source. Script MUST say "fans believe" or "one theory suggests."

RULE: Because this video must not exaggerate, prefer Tier one. Only use Tier three if the WAIT-WHAT score is five AND the script clearly flags it as a theory.

MUSIC QUERY RULE
═══════════════════════════════════
Run these checks IN ORDER. Stop at the first match.

STEP A — FANDOM ANTHEM CHECK (run first, before official OST):
Search whether fans have already adopted a non-official song as the
unofficial theme for this exact character, moment, or matchup —
check fan-edit culture (TikTok edit captions, YouTube AMVs, Know Your
Meme, fan petitions). This is common for iconic 1v1s and viral
character moments (e.g. "Judas" by Lady Gaga for Gojo vs Sukuna,
"Tuca Donka" for Hakari).
  - Confirm dominance: multiple independent high-view edits credit
    the SAME song for the SAME scene, not just one creator's pick.
  - IF a dominant fandom-anthem track exists → set it as music_query.
    Add "is_fan_anthem": true to the output so downstream editors
    know to caption it as fan-favorite, not canon score.

STEP B — OFFICIAL OST CHECK (only if Step A finds nothing dominant):
Decide whether [SERIES] has a recognizable official OST or
instrumental track that matches the script's mood.
  - IF YES → write music_query as before. Set "is_fan_anthem": false.

STEP C — GENERATE (fallback, only if A and B both fail):
Set music_query to null, write music_prompt as before.

TRADE-OFF NOTE: A fandom-anthem track is the most algorithmically
recognizable option, but it's also the MOST overused — thousands of
edits already exist with that exact song. This cuts against the
script's own obscurity/wait-what goals. If the fact itself is the
hook (not the matchup), prefer Step B or C so the audio doesn't feel
like a retread. Use Step A mainly when the visual matchup itself
(not the trivia) is doing the heavy lifting.

── SCENE QUERY (for Gemini / visual search) ──────────────────
scene_query is for an AI visual search tool (e.g. Gemini) to find the single best-matching moment from the series. It is NOT a YouTube search query.

Write scene_query as a pure physical description of what the screen looks like:
- Describe colors, clothing, hair, and objects visible on screen
- Describe kinetic action: dashing, eyes widening, glowing aura, debris falling, steam rising
- No lore jargon. Translate effects into visuals
- Anchor to the most visually surprising moment the script references
- If the script references a specific scene, the scene_query must match that exact scene — do not guess or generalize

── YOUTUBE QUERIES (for sourcing raw footage) ────────────────
Generate exactly six YouTube queries. Write them the way a real fan types into the YouTube search bar — short, natural, casual. Two to six words. NOT keyword-stuffed.

RULES FOR NATURAL QUERIES:
- Use the words fans actually put in titles: "full fight", "scene", "moment", "edit", "vs", "best", "english dub/sub", the character's name, the move's name.
- Drop technical jargon real uploaders never write: avoid "creditless", "isolated footage", "no commentary", "raw", "remaster", "highlight reel" — these rarely appear in actual titles and shrink your results.
- Lead with the most recognizable name: character, attack, or the iconic moment. That is what gets typed and titled most.
- One scene per query set: if the script is about a specific moment, every query chases THAT moment in different words.
- Each of the six must hit a DIFFERENT pool of results, not reword the same search.

THE SIX ANGLES (natural phrasing):

Query one — THE DIRECT MOMENT
What a fan types to find the exact scene.
e.g. "[character] vs [character] full fight" / "[iconic moment] scene"

Query two — THE EMOTIONAL / VIRAL FRAMING
How fans title the moment when it blew up.
e.g. "[character] [moment] goes hard" / "the scene that broke the internet" / "[character] death scene"

Query three — THE EDIT POOL
Short fan edits, usually clean and pre-trimmed.
e.g. "[character] edit" / "[character] [song or vibe] edit"

Query four — EPISODE / ARC POOL
Tied to where it happened, pulls from a different upload set.
e.g. "[series] episode [number] [moment]" / "[arc name] best scene"

Query five — DUB vs SUB POOL
Same scene, different audience, different uploads.
e.g. "[character] [moment] english dub" / "[moment] sub"

Query six — OFFICIAL / CLIP POOL
Studio or official-channel uploads, lowest copyright friction.
e.g. "[series] [moment] official clip" / "[series] official trailer [arc]"

SCENE ACCURACY RULE: If the script references a specific moment, all six queries must chase that exact moment — not the arc, not the character in general.

OUTPUT: Replace every bracket with the real series/character/scene names before outputting. No leftover brackets.
